各种 Normalization 方法详解与 Pre-RMSNorm 深度解析

normalization 技术在深度学习中至关重要，它能加速模型收敛并稳定训练过程。

为了直观理解，我们设定输入数据的张量形状为 (B,C,H,W)(B, C, H, W)(B,C,H,W)：

  • BBB (Batch Size): 批次大小
  • CCC (Channel): 通道数
  • H,WH, WH,W (Height, Width): 空间维度（对于 NLP 任务，可以理解为 LLL 序列长度和 DDD 隐藏层维度）

1. 四大经典归一化方法 (BN, LN, IN, GN)

这四种方法的核心区别在于：计算均值 (μ\muμ) 和方差 (σ2\sigma^2σ2) 时，是在哪些维度上进行的。

(1) Batch Normalization (BN)

  • 核心逻辑：对 整个 Batch 的同一个通道进行归一化。它假设同一个通道的特征在不同样本间具有可比性。
  • 计算范围：在 (B,H,W)(B, H, W)(B,H,W) 维度上聚合。
  • 一次计算的数据量：对于每个通道 ccc，计算 B×H×WB \times H \times WB×H×W 个数据的统计量。
  • 特点：
    • 优点：显著加速收敛，对初始化不敏感。
    • 缺点：强烈依赖 Batch Size，当 BS 较小（如 < 8）时效果极差；且在 RNN/Transformer 等变长序列中很难使用。
    • 公式：

      x^b,c,h,w=xb,c,h,w−μcσc2+ϵ⋅γc+βc\hat{x}_{b,c,h,w} = \frac{x_{b,c,h,w} - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}} \cdot \gamma_c + \beta_c x^b,c,h,w​=σc2​+ϵ​xb,c,h,w​−μc​​⋅γc​+βc​

(2) Layer Normalization (LN)

  • 核心逻辑：对 单个样本 的所有通道进行归一化。它假设同一个样本内的不同通道共同构成了该样本的“语义”，不依赖其他样本。
  • 计算范围：在 (C,H,W)(C, H, W)(C,H,W) 维度上聚合。
  • 一次计算的数据量：对于每个样本 bbb，计算 C×H×WC \times H \times WC×H×W 个数据的统计量。
  • 特点：
    • 优点：完全独立于 Batch Size，非常适合 RNN 和 Transformer。
    • 缺点：在 CNN（计算机视觉）中效果通常不如 BN，因为不同通道（如边缘检测器 vs 纹理检测器）的分布差异很大，强行一起归一化可能会破坏特征。

(3) Instance Normalization (IN)

  • 核心逻辑：对 单个样本 的 单个通道 进行归一化。
  • 计算范围：在 (H,W)(H, W)(H,W) 维度上聚合。
  • 一次计算的数据量：对于每个样本 bbb 的每个通道 ccc，计算 H×WH \times WH×W 个数据的统计量。
  • 特点：
    • 主要场景：风格迁移（Style Transfer）、GAN。因为它过滤掉了图片的“对比度”和“亮度”等全局风格信息，只保留内容结构。

(4) Group Normalization (GN)

  • 核心逻辑：LN 和 IN 的折中方案。将 CCC 个通道分为 GGG 个组，在每组内进行归一化。
  • 计算范围：在 (C/G,H,W)(C/G, H, W)(C/G,H,W) 维度上聚合。
  • 一次计算的数据量：对于每个样本 bbb 的每个组 ggg，计算 (C/G)×H×W(C/G) \times H \times W(C/G)×H×W 个数据的统计量。
  • 特点：
    • 优点：解决了 BN 对 Batch Size 的依赖，同时保留了部分通道间的依赖关系。在检测、分割等小 Batch 任务中表现优异。

2. Pre-RMSNorm (Pre-Layer Root Mean Square Normalization)

这是目前大语言模型（如 LLaMA、Gemma）中最主流的配置。它实际上包含两个概念的结合：RMSNorm（算法） 和 Pre-Norm（位置）。

A. 算法核心：RMSNorm (Root Mean Square Norm)

RMSNorm 是 Layer Norm 的一种简化变体。传统的 LN 需要计算均值 μ\muμ 和方差 σ2\sigma^2σ2（即由 Center 和 Scale 两步组成）。
RMSNorm 假设：不需要 Center（减均值），只需要 Scale（除以均方根）。

  • 公式：
    假设输入向量为 xxx，维度为 ddd（对应上面的 CCC）：

    RMS(x)=x1d∑i=1dxi2+ϵ⋅γ\text{RMS}(x) = \frac{x}{\sqrt{\frac{1}{d} \sum_{i=1}^{d} x_i^2 + \epsilon}} \cdot \gamma RMS(x)=d1​∑i=1d​xi2​+ϵ​x​⋅γ

    注意：这里只有缩放参数 γ\gammaγ，没有平移参数 β\betaβ。

  • 为什么用 RMSNorm？

    1. 计算效率：少算一个均值，计算速度更快（在超大规模 LLM 训练中节省的时间很可观）。
    2. 效果相当：研究表明，Layer Norm 的成功主要归功于缩放（Rescaling）带来的梯度稳定性，而非平移（Re-centering）。

B. 结构位置：Pre-Norm vs. Post-Norm

这指的是 Normalization 层在 Transformer Block 中的位置。

  • Post-Norm (原始 BERT/Transformer)：

    xout=Norm(x+Sublayer(x))x_{out} = \text{Norm}(x + \text{Sublayer}(x)) xout​=Norm(x+Sublayer(x))

    先做 Attention/FFN，残差连接后再做 Norm。

    • 问题：梯度在反向传播时容易在靠近输出层的地方很大，在靠近输入层的地方消失，导致深层模型很难训练（需要 Warmup 技巧）。

  • Pre-Norm (LLaMA/GPT-3)：

    xout=x+Sublayer(Norm(x))x_{out} = x + \text{Sublayer}(\text{Norm}(x)) xout​=x+Sublayer(Norm(x))

    在进入 Attention/FFN 之前 先做 Norm，残差连接路径上是“干净”的（没有 Norm 层阻断）。

    • 优势：梯度高速公路。梯度可以直接通过恒等映射（Identity path）传到底层，极大地稳定了深层网络的训练，允许训练更深的模型。

C. Pre-RMSNorm 总结

Pre-RMSNorm 就是将 RMSNorm 放在残差块的“输入端”。

  • 对于输入 (B,L,D)(B, L, D)(B,L,D)（LLM 常见维度）：
    • 计算范围：针对每个 Token 的 Embedding 向量（维度 DDD）计算 RMS。
    • 数据量：一次计算 DDD 个数值的平方和。
    • 优势：结合了 Pre-Norm 的训练稳定性（Deep Transformer 不易梯度爆炸/消失）和 RMSNorm 的计算高效性。

3. 总结对比表 (假设输入 B,C,H,WB, C, H, WB,C,H,W)

方法 归一化维度 (聚合轴) 统计量计算范围 (像素数) 依赖 Batch Size? 典型应用场景
BN (B,H,W)(B, H, W)(B,H,W) B×H×WB \times H \times WB×H×W 是 (严重) CNN, ResNet (CV)
LN (C,H,W)(C, H, W)(C,H,W) C×H×WC \times H \times WC×H×W 否 Transformer, BERT, RNN (NLP)
IN (H,W)(H, W)(H,W) H×WH \times WH×W 否 风格迁移, GAN
GN (C/G,H,W)(C/G, H, W)(C/G,H,W) (C/G)×H×W(C/G) \times H \times W(C/G)×H×W 否 物体检测, 分割 (小 Batch)
RMSNorm (C,H,W)(C, H, W)(C,H,W) 或 (D)(D)(D) 同 LN，但不减均值 否 LLM (LLaMA, GPT-3)

给算法工程师的建议：
如果你在做 CV 任务且显存充足（BS 大），首选 BN。如果是 NLP 或 LLM 任务，毫无疑问选择 Pre-RMSNorm。如果是小 Batch 的 CV 任务（如医疗影像分割），GN 是最佳替补。
