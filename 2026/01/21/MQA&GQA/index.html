MQA (Multi-Query Attention) 与 GQA (Grouped-Query Attention)

第一部分：核心原理与公式推导

这两种机制主要是为了解决传统 MHA (Multi-Head Attention) 在大规模模型推理（Inference）时的显存占用和带宽瓶颈问题而提出的。

1. 背景：标准多头注意力 (Multi-Head Attention, MHA)

在深入 MQA 和 GQA 之前，我们需要先回顾一下标准的 MHA，作为对比基准。

在 Transformer 的 MHA 中，输入序列被映射为 Query (QQQ)、Key (KKK) 和 Value (VVV)。假设模型有 HHH 个注意力头（Heads），每个头的维度为 dheadd_{head}dhead​。

  • 机制： 每个头都有自己独立的 Wq,Wk,WvW_q, W_k, W_vWq​,Wk​,Wv​ 投影矩阵。这意味着对于每个 Query 头，都有一组对应的 Key 和 Value 头。
  • 公式：

    Headi=Attention(QWiQ,KWiK,VWiV)\text{Head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V) Headi​=Attention(QWiQ​,KWiK​,VWiV​)

    MultiHead(Q,K,V)=Concat(Head1,...,HeadH)WO\text{MultiHead}(Q, K, V) = \text{Concat}(\text{Head}_1, ..., \text{Head}_H) W^O MultiHead(Q,K,V)=Concat(Head1​,...,HeadH​)WO

  • 推理瓶颈： 在生成式任务（如 LLM 的解码阶段）中，为了加速，我们通常会缓存 KKK 和 VVV 的计算结果（即 KV Cache）。由于 MHA 中 KKK 和 VVV 的头数与 QQQ 一样多（HHH 个），随着序列长度增加，KV Cache 占用的显存会线性急剧增长，导致显存带宽成为推理速度的瓶颈（Memory Wall）。

2. Multi-Query Attention (MQA)

MQA 最早由 Google 在 2019 年提出（论文：Fast Transformer Decoding: One Write-Head is All You Need）。

2.1 核心原理

MQA 的核心思想非常极致：所有的 Query 头共享同一组 Key 和 Value 头。

  • MHA: HHH 个 Query 头，对应 HHH 个 Key 头和 HHH 个 Value 头。
  • MQA: HHH 个 Query 头，对应 1 个 Key 头和 1 个 Value 头。

这意味着在进行注意力计算时，所有的 Query 向量都与同一个 K,VK, VK,V 矩阵进行交互。

2.2 公式推导与维度变化

假设 Batch Size 为 BBB，序列长度为 LLL，头数为 HHH，头维度为 DDD。

  • MHA 的张量形状：

    • QQQ: (B,H,L,D)(B, H, L, D)(B,H,L,D)
    • KKK: (B,H,L,D)(B, H, L, D)(B,H,L,D)
    • VVV: (B,H,L,D)(B, H, L, D)(B,H,L,D)
    • KV Cache 大小 ∝2×H×L×D\propto 2 \times H \times L \times D∝2×H×L×D

  • MQA 的张量形状：

    • QQQ: (B,H,L,D)(B, H, L, D)(B,H,L,D)
    • KKK: (B,1,L,D)(B, 1, L, D)(B,1,L,D) ←\leftarrow← 注意这里的 1
    • VVV: (B,1,L,D)(B, 1, L, D)(B,1,L,D) ←\leftarrow← 注意这里的 1
    • KV Cache 大小 ∝2×1×L×D\propto 2 \times 1 \times L \times D∝2×1×L×D

在计算 Attention Score 时，需要对 KKK 和 VVV 进行 Broadcasting（广播） 操作，使其在 Head 维度上复制 HHH 次（或者在逻辑上等价于复制），以匹配 QQQ 的维度。

Attention(Qi,K,V)=softmax(QiKTdhead)V\text{Attention}(Q_i, K, V) = \text{softmax}\left(\frac{Q_i K^T}{\sqrt{d_{head}}}\right) V Attention(Qi​,K,V)=softmax(dhead​​Qi​KT​)V

其中 QiQ_iQi​ 是第 iii 个头的 Query，而 K,VK, VK,V 对所有 iii 都是共享的。

2.3 优缺点

  • 优点： 极大地减少了 KV Cache 的显存占用（减少为原来的 1/H1/H1/H），显著降低了推理时的内存带宽需求，大幅提升 Decoding 速度。
  • 缺点： 由于过度压缩了 KKK 和 VVV 的信息，模型的表达能力会受到一定损失，可能导致生成质量（Perplexity）下降，且训练稳定性可能不如 MHA。

3. Grouped-Query Attention (GQA)

GQA 由 Google 在 2023 年提出（论文：GQA: Training Generalized Multi-Query Attention），并在 LLaMA-2/3 等模型中被广泛采用。它是 MHA 和 MQA 的折中方案。

3.1 核心原理

GQA 将 Query 头分成 GGG 个组（Group），每个组内的 Query 头共享同一组 Key 和 Value 头。

  • 设总头数为 HHH，组数为 GGG。
  • 每个组包含 H/GH/GH/G 个 Query 头。
  • 整个模型共有 GGG 个 Key 头和 GGG 个 Value 头。

特例关系：

  • 当 G=1G=1G=1 时，GQA 退化为 MQA。
  • 当 G=HG=HG=H 时，GQA 退化为 MHA。

3.2 公式推导

假设我们将 Query 分为 GGG 组，每组包含 m=H/Gm = H/Gm=H/G 个子头。
对于第 ggg 组（g∈{1,...,G}g \in \{1, ..., G\}g∈{1,...,G}）：

  • 该组有一对专属的 Kg,VgK_g, V_gKg​,Vg​。
  • 该组包含 Qg,1,Qg,2,...,Qg,mQ_{g,1}, Q_{g,2}, ..., Q_{g,m}Qg,1​,Qg,2​,...,Qg,m​。

计算公式为：

Headg,j=Attention(Qg,j,Kg,Vg)\text{Head}_{g, j} = \text{Attention}(Q_{g,j}, K_g, V_g) Headg,j​=Attention(Qg,j​,Kg​,Vg​)

其中 jjj 表示组内的第 jjj 个 Query 头。在计算时，KgK_gKg​ 和 VgV_gVg​ 同样需要通过 Broadcasting 扩展到与组内 Query 数量一致。

3.3 维度对比表

为了更直观地理解，假设 H=8H=8H=8：

机制 Query 头数量 Key/Value 头数量 比例 (Q:KV) 特点
MHA 8 8 1 : 1 质量最高，显存占用最大，推理慢
GQA (Group=2) 8 4 2 : 1 质量与速度的平衡
GQA (Group=4) 8 2 4 : 1 更接近 MQA
MQA 8 1 8 : 1 速度最快，显存最小，质量略降

3.4 优缺点

  • 优点： GQA 在几乎保持 MHA 级别的模型效果的同时，能够获得接近 MQA 的推理速度和低显存占用。它提供了一个可调节的旋钮（组数 GGG），让设计者在速度和质量之间通过实验找到最佳平衡点。
  • 应用： 目前主流的大模型（如 LLaMA-3 70B）普遍采用 GQA。

第二部分：架构差异与性能深度解析（进阶篇）

本部分基于后续讨论，对架构图的物理含义、计算量（FLOPs）的真实变化以及性能瓶颈进行了深入拆解。

1. 架构映射关系的修正

理解 MQA/GQA 的关键在于理解 “Query 多，Key/Value 少” 的非对称结构。

  • MHA (1:1 映射):

    • 物理结构：8 个 Query 头，8 个 Key 头。
    • 逻辑：每一个 Q 都有其专属的 K。

  • GQA (N:M 映射，例如 2:1):

    • 物理结构：8 个 Query 头，4 个 Key 头。
    • 逻辑：每 2 个 Q 共享 1 个 K。计算前，这 1 个 K 会被复制（Broadcast）2 次来匹配 Q。

  • MQA (N:1 映射):

    • 物理结构：8 个 Query 头，1 个 Key 头。
    • 逻辑：所有 8 个 Q 共享这 1 个 K。

2. 计算量（FLOPs）到底变没变？

MQA/GQA 的性能提升主要来自显存带宽优化，但在计算量（FLOPs）上也有具体体现。我们将 Attention 操作拆分为两个阶段分析：

阶段一：线性投影 (Linear Projection) —— FLOPs 显著下降

这是从隐藏层输入 XXX 生成 Q,K,VQ, K, VQ,K,V 的过程（矩阵乘法）。

  • 原理： MQA/GQA 中，由于 KKK 和 VVV 的头数减少，对应的权重矩阵 WKW_KWK​ 和 WVW_VWV​ 的尺寸也变小了。
  • 数值案例：
    假设 dmodel=4096d_{model} = 4096dmodel​=4096, H=32H = 32H=32, dhead=128d_{head} = 128dhead​=128。
    • MHA: 需要计算 32 个头的 K。
      计算量 ≈4096×(32×128)≈1677\approx 4096 \times (32 \times 128) \approx 1677≈4096×(32×128)≈1677 万次乘法。
    • MQA: 只需要计算 1 个头的 K。
      计算量 ≈4096×(1×128)≈52\approx 4096 \times (1 \times 128) \approx 52≈4096×(1×128)≈52 万次乘法。
    • 结论： 在生成 K 和 V 的投影阶段，MQA 的计算量约为 MHA 的 1/H。

阶段二：注意力核心计算 (Core Attention) —— FLOPs 基本不变

这是计算 Attention=softmax(QKT)VAttention = \text{softmax}(QK^T)VAttention=softmax(QKT)V 的过程。

  • 原理： 虽然 K,VK, VK,V 的物理存储变少了，但在计算点积时，逻辑上必须将 KKK 扩展（Broadcast）到与 QQQ 相同的头数。
  • 结论： 最终的点积次数取决于 Query 的头数。只要 Query 头数没变（例如还是 32 个），这一步的矩阵乘法次数几乎没有变化。

3. 真正的性能瓶颈：IO Bound vs Compute Bound

为什么 MQA/GQA 能让推理速度快那么多？

  • LLM 推理特点： 解码（Decoding）阶段是逐个 Token 生成的，这是一个典型的 Memory-Bound（内存受限） 场景。
  • 瓶颈所在： GPU 的计算单元（Tensor Core）通常在等待显存（HBM）把庞大的 KV Cache 数据搬运过来。计算本身很快，搬运数据很慢。
  • 优化本质： * MHA 需要搬运巨大的 KV Cache。
    • MQA/GQA 极大地压缩了 KV Cache 的体积（例如 MQA 压缩了 32 倍）。
    • 结果： 显存带宽压力骤减，数据搬运时间大幅缩短，从而显著提升了 Token 生成速度（Tokens per second）。

总结

MQA 和 GQA 是在模型质量（Quality）和推理效率（Efficiency）之间做权衡（Trade-off）的产物。它们通过减少参数量降低了投影层的计算量，通过压缩 KV Cache 解决了推理时的显存墙（Memory Wall）问题。

MQA/GQA 并非在 Attention 算法本身做了数学创新，而是通过缩减 WKW_KWK​ 和 WVW_VWV​ 权重矩阵的宽度，实现了对投影层输出（KV Cache）的硬性压缩。这种压缩牺牲了微小的建模精度，换取了推理阶段巨大的内存带宽红利。
