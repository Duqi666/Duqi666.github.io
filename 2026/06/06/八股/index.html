一、 Attention 与 Transformer 架构

1、Attention 计算公式：Attention为什么要 scaled（为什么除以根号dk\sqrt{d_k}dk​​）？不做会怎样？除了除以dk\sqrt{d_k}dk​​还有什么办法？

原因：
主要是为了防止梯度消失，维持训练的稳定性
在 Transformer 架构中，Attention 的计算公式为：

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V Attention(Q,K,V)=softmax(dk​​QKT​)V

假设输入向量 QQQ 和 KKK 的每个元素都是独立同分布的随机变量，且满足均值为 0，方差为 1（即符合标准正态分布：X∼N(0,1)X \sim N(0, 1)X∼N(0,1)），每个向量的维度是 dkd_kdk​，那么 QKTQK^TQKT 的每个元素的均值为 0，方差为 dkd_kdk​。将这个结果除以dk\sqrt{d_k}dk​​可以将点积的方差重新拉回到了 1，保证了无论模型维度（如 1024、2048 甚至更大）怎么变，Softmax 输入的方差始终稳定。
不做会怎样？
如果不除以 dk\sqrt{d_k}dk​​，当 dkd_kdk​ 很大时，内积 QKTQK^TQKT 的方差会非常大。这会导致某些位置的得分极大，而另一些位置的得分极小。Softmax 的输出会极其接近一个 One-hot 向量（最大值变成 1，其余全变成 0）。反向传播时，Attention 权重的梯度几乎为零，模型根本无法学到有效的特征连接，训练直接瘫痪或无法收敛。
除了除以 dk\sqrt{d_k}dk​​，还有什么办法？

  • 更好的归一化：Layer Normalization 位置调整，在经典 Transformer 中采用的是 Post-LN（对残差相加后的结果做 LN）。现在大模型（如 Llama, Mistral）标配 Pre-LN（在输入 Attention 前先做 RMSNorm 或 LayerNorm）。通过在计算 Q,KQ, KQ,K 之前就严格限制住输入的方差，从源头上遏制了内积过大的趋势。但为了保险，代码里依然会保留 dk\sqrt{d_k}dk​​。
  • 用其他激活函数替代 Softmax
    • Squared ReLU：Primer 论文中提出在 Attention 中使用 ReLU2\text{ReLU}^2ReLU2 替代 Softmax。它没有指数爆炸问题，不需要显式的 dk\sqrt{d_k}dk​​ 缩放，且在某些大模型训练中收敛更快。
    • 点积归一化（Cosine Similarity Attention）：不用内积，改用余弦相似度：$$\text{Attention Score} = \text{softmax}(\gamma \cdot \cos(Q, K))$$因为余弦相似度的值域天生被限制在 [−1,1][-1, 1][−1,1] 之间，完全不需要担心维度 dkd_kdk​ 带来的爆炸问题（只需要乘一个可学习的温度系数 γ\gammaγ 即可）。
    • Mamba / RWKV / Linear Attention：这些架构通过移除传统的 Softmax 矩阵映射，将其转化为类似 RNN 的线性动态系统。它们不计算全量两两相乘的 Attention 矩阵，因此从本质上规避了 dk\sqrt{d_k}dk​​ 的困扰。

二、 强化学习 (RL) 训练流程

PPO 训练细节

1、PPO 如何做的详细介绍，包括数据，怎么训练奖励模型，每个模型的损失什么样子的？

详细介绍
从四个模型和整体的目标函数介绍

数据

  • 数据准备：直接进行prompt采样，不需要标准答案
  • 数据流转步骤：
    • 目标模型针对prompt生成回答；
    • 调用奖励函数打分，并加入相关奖励修正，例如当前模型与reference model分布的KL散度，计算出每一个token的奖励分；
    • 调用Critic模型计算Advantages；

奖励函数的训练方式
在开启 PPO 之前，必须先训练好 RM。RM 结构上是将 SFT 模型的最后的 LM Head（Token 分类器）替换成一个一维的 Value Head，从而输出一个标量。

  • 训练数据
    • Pairwise 偏好数据：格式为 (x,yw,yl)(x, y_w, y_l)(x,yw​,yl​)。xxx 是 Prompt，ywy_wyw​ 是人类偏好的优质回答（Chosen），yly_lyl​ 是被拒绝的低质量回答（Rejected）。
  • RM 损失函数（Pairwise Ranking Loss）
    • RM 的目标是让 ywy_wyw​ 的得分尽可能高于 yly_lyl​ 的得分，采用交叉熵排序损失：$$\mathcal{L}{\text{RM}}(\theta) = -\mathbb{E}{(x, y_w, y_l) \sim D} \left[ \log \sigma \left( r_\theta(x, y_w) - r_\theta(x, y_l) \right) \right]$$

PPO 阶段核心模型的损失函数（Loss）
在 PPO 的更新阶段，主要对 Actor 和 Critic 两个更新梯度的模型计算 Loss：

  • Actor Model Loss（策略损失）

LActor(ϕ)=−Et[min⁡(rt(ϕ)At, clip(rt(ϕ),1−ϵ,1+ϵ)At)]\mathcal{L}_{\text{Actor}}(\phi) = - \mathbb{E}_{t} \left[ \min \left( r_t(\phi) A_t, \, \text{clip}(r_t(\phi), 1-\epsilon, 1+\epsilon) A_t \right) \right] LActor​(ϕ)=−Et​[min(rt​(ϕ)At​,clip(rt​(ϕ),1−ϵ,1+ϵ)At​)]

  • Critic Model Loss（价值损失）
    Critic 的目标是让预测的 Vψ(st)V_\psi(s_t)Vψ​(st​) 尽可能逼近由 GAE 算出来的真实目标回报（Target Value）$V_t^{target} $。通常基于均方误差（MSE）计算：

LVF(ϕ)=12Et[(V^ϕ(st)−Vttarget)2]L^{VF}(\phi) = \frac{1}{2} \mathbb{E}_t \left[ \left( \hat{V}_\phi(s_t) - V_t^{target} \right)^2 \right] LVF(ϕ)=21​Et​[(V^ϕ​(st​)−Vttarget​)2]

其中：$$V_t^{target} = A_t^{GAE} + \hat{V}{\phi{old}}(s_t)$$

为什么要用 reference model？为了解决什么问题？

在计算每个token的得分时，会加入当前模型与reference模型的KL散度惩罚

Rewardfinal=Rewardmodel(s,a)−βlog⁡πθ(a∣s)πref(a∣s)\text{Reward}_{final} = \text{Reward}_{model}(s, a) - \beta \log \frac{\pi_\theta(a|s)}{\pi_{ref}(a|s)} Rewardfinal​=Rewardmodel​(s,a)−βlogπref​(a∣s)πθ​(a∣s)​

主要的目的是：

  • 为了防止模型利用 Reward Model 的盲区去刷分（Reward Hacking）
  • 为了防止模型学了新规矩忘了旧知识（灾难性遗忘）
  • 为了防止模型在极高维的空间里训练崩塌

我们必须引入 Reference Model 作为“理性的化身”，随时把跑偏的当前模型拉回正轨。
