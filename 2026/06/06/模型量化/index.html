模型量化-PTQ和QAT

Created2026-06-06|Updated2026-06-07|大模型优化

引言

模型量化是指将模型的参数从高精度的浮点数转换为低精度的整数，以减少模型的参数量和计算量。

先来看各种数据格式的精度：

  • 符号位：表示正负号
  • 指数位：表示数值的范围，指数会表征两个数字：
    • 偏置：是一个基准，Bias=2E−1−1Bias = 2^{E-1} - 1Bias=2E−1−1
    • 指数位的实际值

      • 范围：Actualmin=000012=1Actual_min = 00001_2 = 1Actualm​in=000012​=1，Actualmax=111102=31Actual_max = 11110_2 = 31Actualm​ax=111102​=31，因为全0和全1表示特殊值，所以范围是[1,31][1, 31][1,31]，那么取值范围就是[2−14,215][2^{-14}, 2^{15}][2−14,215]
  • 尾数位：表示数值的精度
    • 例如对于E5M2，尾数位是2位，加入为10，这里第一位的1是隐藏的，那么实际尾数 = 1+二进制的 0.10=1+12+04=1+0.5=1.51 + \text{二进制的 } 0.10 = 1 + \frac{1}{2} + \frac{0}{4} = 1 + 0.5 = 1.51+二进制的 0.10=1+21​+40​=1+0.5=1.5
    • 所以实际值是1.5×27=1921.5 \times 2^7 = 1921.5×27=192

PTQ（Post-Training Quantization，训练后量化）

不需要重新训练模型，只需少量样本（如 128 条）计算校准集（Calibration Layer），直接生成量化权重。直接将大模型的参数量化，即可实现模型的量化。大模型普遍采用 PTQ，因为微调成本太高。

PTQ 的数学底层原理

量化的数学本质，就是把一个大范围的浮点数 xxx 映射到一个小范围的整数 qqq。最常见的线性量化（Uniform Quantization）公式如下：
量化（Quantize）

q=clip(round(xScale)+ZeroPoint,qmin,qmax)q = \text{clip}\left( \text{round}\left( \frac{x}{\text{Scale}} \right) + \text{ZeroPoint}, \quad q_{min}, \quad q_{max} \right) q=clip(round(Scalex​)+ZeroPoint,qmin​,qmax​)

  • Scale（缩放因子）： 决定了粗刻度尺子的“格子间距”。是一个浮点数。
  • ZeroPoint（零点）： 用来保证浮点数中的 0 映射到整数时也是一个精准的整数，以便在 Padding 等操作中不引入误差（如果是对称量化，ZeroPoint 则固定为 0）。
  • round()\text{round}()round()： 四舍五入取整。
  • 裁剪（Clip）： 如果算出来的 qqq 超出了低比特的边界（比如 INT4 的 −8∼7-8 \sim 7−8∼7），就硬截断到边界值。

量化又分为对称量化和非对称量化：

非对称量化

  • scale的计算方式：

Scale=β−αqmax−qmin\text{Scale} = \frac{\beta - \alpha}{q_{max} - q_{min}} Scale=qmax​−qmin​β−α​

其中：

α=min⁡(X),β=max⁡(X)\alpha = \min(X), \quad \beta = \max(X) α=min(X),β=max(X)

这里需要注意的是，α\alphaα 和 β\betaβ 是实际校准集 XXX 的最小值和最大值，而不是这种数据格式的固有输入范围。

  • 计算 ZeroPoint
    ZeroPoint 的目的是让浮点数里的 0.00.00.0 能被精准表示（原始的0，量化之后再反量化必须变为0）。我们通过把 α\alphaα 映射到 qminq_{min}qmin​ 的线性公式反推：

ZeroPoint=round(−αScale)+qmin\text{ZeroPoint} = \text{round}\left( \frac{-\alpha}{\text{Scale}} \right) + q_{min} ZeroPoint=round(Scale−α​)+qmin​

对称量化

Scale=γqmax\text{Scale} = \frac{\gamma}{q_{max}} Scale=qmax​γ​

其中

γ=max⁡(∣X∣)\gamma = \max(|X|) γ=max(∣X∣)

ZeroPoint=0\text{ZeroPoint} = 0 ZeroPoint=0

思考：所以这里的α\alphaα 和 β\betaβ的确定是很有说法的，如果直接取校准集中的全局最大最小值，那么很有可能被离群点干扰导致分布不均匀，所以需要在大范围覆盖与clip取舍之间做平衡。

反量化（Dequantize）
在计算或者输出时，我们需要把整数还原回去：

x^=(q−ZeroPoint)×Scale\hat{x} = (q - \text{ZeroPoint}) \times \text{Scale} x^=(q−ZeroPoint)×Scale

PTQ 的核心工作流程

PTQ 并不是简单地对模型参数进行盲目的“四舍五入”，它通常包含以下四个核心步骤：

  • 准备校准集（Calibration Set）： 从通用的文本数据中随机抽取极少量的样本（通常只需要 32 到 128 条 文本）。

  • 前向传播与数据统计（Activation Calibration）： 将校准集输入模型进行前向传播。在这个过程中，算法会像一个“监控探头”一样，记录每一层网络中激活值（Activations）的动态范围（即最大值、最小值和整体分布）。

  • 计算缩放因子（Scale & Zero-Point）： 根据统计到的权重和激活值分布，利用特定的数学算法（如下文提到的经典算法）寻找最佳的映射尺子。

  • 转换与存储： 利用计算好的 Scale 将模型参数真正截断为低比特（如 INT4/INT8），并打包导出。

其实简单来说就是拿一小部分数据输入到模型中跑一下，记录下每一层网络中激活值的动态范围，即α\alphaα 和 β\betaβ，然后根据这个范围来计算缩放因子和零点，最后将模型参数截断为低比特的整数。

QAT（Quantization-Aware Training，量化感知训练）

核心思想是：把量化带来的“四舍五入误差（Round Error）”，直接放在模型的训练循环（Training Loop）里，让模型在训练时自己去适应这种模糊感。

伪量化（Fake Quantization）

在真正的硬件部署中，INT4 的矩阵乘法是不支持反向传播和梯度更新的（因为整数没有细微的梯度微调概念）。因此，QAT 在训练时，网络里的所有权重（Weights）和激活值（Activations）在骨子里依然是高精度的浮点数（FP32 或 BF16），只是在数值上“假装”被量化了。

为了实现这一点，QAT 在每一个需要量化的算子前，都会强行插入一个伪量化节点（Fake Quant Node）。

前向传播：故意将权重经过量化+反量化操作，引入误差

当前向传播的矩阵乘法发生前，浮点权重 WWW 会被迫在伪量化节点里走一遭：

  • 真量化：q=clip(round(WScale)+ZP, qmin, qmax)q = \text{clip}\left(\text{round}\left(\frac{W}{\text{Scale}}\right) + \text{ZP}, \ q_{min}, \ q_{max}\right)q=clip(round(ScaleW​)+ZP, qmin​, qmax​) （此时变成了纯整数，比如 12）
  • 立即反量化：Wfake=(q−ZP)×ScaleW_{\text{fake}} = (q - \text{ZP}) \times \text{Scale}Wfake​=(q−ZP)×Scale （重新变回浮点数，比如 11.875）

经过这一变，原本高精度的 W=11.9213W = 11.9213W=11.9213 变成了 Wfake=11.875W_{\text{fake}} = 11.875Wfake​=11.875。中间损失掉的 0.04630.04630.0463 就是量化带来的噪点（Quantization Noise）。 网络拿着这个变模糊的 WfakeW_{\text{fake}}Wfake​ 去做矩阵乘法，算出最终的 Loss。

反向传播与 STE 机制：
反向传播时，由于量化和反量化过程中的round算子不可导，则使用STE直接传递梯度

训练和推理流程

这里需要先了解QAT的两种方案：

  • 仅权重量化 (Weight-Only)
    • 只将模型中静态的权重（Weights）压缩为低比特整数（如 INT4），而动态的激活值（Activations）在推理时全程保持高精度的浮点格式（如 FP16/BF16）。这种方案的本质是针对大模型生成文本时的“显存带宽瓶颈（Memory-Bound）”进行极致优化，在推理计算的瞬间将整数权重在线反量化回浮点数进行常规乘法，从而用极低的精度损失白嫖到近 4 倍的显存缩减与成倍的吐字速度提升。
  • 权重激活全量化 (Weight-Activation)
    • 不仅把静态的权重压缩成低比特，连网络运行时动态产生的激活值也一起切成低比特（如 INT8）。这种方案的终极目标是彻底解放 GPU 或专用 AI 芯片的低比特计算算力（“计算瓶颈，Compute-Bound”），让整数权重与整数激活值在硬件底层的 Tensor Core 中直接硬碰硬地跑纯整数矩阵乘法（无需中间在线反量化），从而在面对高并发、高吞吐的工业级大模型服务时，展现出极其恐怖的算力爆发和极低的首字延迟。

针对上述两种方案，伪量化算子的实现如下：

  • 针对仅权重量化方案，伪量化算子的只在权重上进行；
  • 针对权重激活全量化方案，伪量化算子需要分别在权重和激活值上进行，以保持模型的动态计算能力。

训练

  1. 前向传播
    • 仅权重量化 (Weight-Only)：将权重量化为低比特整数，再反量化为高精度浮点数；
    • 权重激活全量化 (Weight-Activation)：将权重量和激活值都化为低比特整数，再反量化为高精度浮点数。
  2. 反向传播
    • 使用STE直接传递梯度，更新权重。

存储

  1. 存储模型参数时，将权重量化为低比特整数减小存储。

推理

  • 仅权重量化 (Weight-Only)：在推理时，将权重量在线反量化回浮点数，再进行矩阵乘法。
  • 权重激活全量化 (Weight-Activation)：在推理时，将权重量和激活值不需要在线反量化回浮点数，直接进行矩阵乘法，然后对结果进行反量化。
