GRPO: Group Relative Policy Optimization 算法详解

本文档深入探讨 DeepSeek-R1 背后的关键强化学习算法——GRPO。我们将从传统的 PPO 算法痛点出发，详细解析 GRPO 的原理、数学推导、代码实现，并通过一个生动的大模型训练案例来演示其完整流程。

1. 背景回顾：PPO 的辉煌与痛点

在介绍 GRPO 之前，我们必须先回顾一下它的前辈——PPO (Proximal Policy Optimization)。PPO 是 ChatGPT 等大模型 RLHF 阶段的“版本答案”，但它并非完美。

1.1 PPO 的核心逻辑 (Actor-Critic)

PPO 依赖于 Actor-Critic (AC) 架构：

  • Actor (策略网络 πθ\pi_\thetaπθ​)：负责生成动作（Token）。
  • Critic (价值网络 VϕV_\phiVϕ​)：负责给状态打分，充当“基线 (Baseline)”。

优势函数 (Advantage) 的计算依赖于 Critic：

At=rt+γV(st+1)−V(st)A_t = r_t + \gamma V(s_{t+1}) - V(s_t) At​=rt​+γV(st+1​)−V(st​)

或者使用更复杂的 GAE (Generalized Advantage Estimation)。

1.2 PPO 在大模型时代的痛点

在训练 7B、70B 甚至更大的 LLM 时，PPO 显得极其“沉重”：

  1. 显存爆炸：我们需要在显存中同时维护 4 个大模型：
    • Actor (训练中)
    • Ref Model (冻结)
    • Reward Model (冻结)
    • Critic (训练中) ←\leftarrow← 最昂贵的累赘
  2. Critic 难训练：Critic 需要理解复杂的语义才能给出准确的估值。如果 Critic 练不好，给出的优势 AtA_tAt​ 就是噪声，导致 Actor 训练崩溃。

GRPO 的诞生原因：能不能去掉 Critic，直接利用同一批次生成的多个样本来计算基线？

2. GRPO 算法详解

GRPO (Group Relative Policy Optimization) 是一种去中心化的、无需 Critic 的策略优化算法。它通过组内归一化来计算优势。

2.2 算法步骤

  1. 组采样 (Group Sampling)：
    对于每一个输入 Prompt qqq，让模型采样生成 GGG 个不同的输出 {o1,o2,...,oG}\{o_1, o_2, ..., o_G\}{o1​,o2​,...,oG​}。

    • 通常 GGG 取值在 4 到 64 之间。

  2. 打分 (Reward Calculation)：
    使用 Reward Model 对这 GGG 个输出进行打分，得到奖励集合 {r1,r2,...,rG}\{r_1, r_2, ..., r_G\}{r1​,r2​,...,rG​}。

  3. 计算相对优势 (Relative Advantage)：
    不使用 Critic 预测的 V(s)V(s)V(s) 作为基线，而是计算这组奖励的平均值和标准差：

    mean=1G∑i=1Gri,std=1G∑i=1G(ri−mean)2\text{mean} = \frac{1}{G}\sum_{i=1}^G r_i, \quad \text{std} = \sqrt{\frac{1}{G}\sum_{i=1}^G (r_i - \text{mean})^2} mean=G1​i=1∑G​ri​,std=G1​i=1∑G​(ri​−mean)2​

    每个输出 oio_ioi​ 的优势 AiA_iAi​ 为：

    Ai=ri−meanstd+ϵA_i = \frac{r_i - \text{mean}}{\text{std} + \epsilon} Ai​=std+ϵri​−mean​

  4. 策略更新 (PPO-Clip Update)：
    利用计算出的 AiA_iAi​，套用 PPO 的 Clip Loss 公式进行更新：

    LGRPO=E[min⁡(πθ(oi∣q)πold(oi∣q)Ai,clip(… )Ai)−βDKL]L_{GRPO} = \mathbb{E} \left[ \min \left( \frac{\pi_\theta(o_i|q)}{\pi_{old}(o_i|q)} A_i, \text{clip}(\dots) A_i \right) - \beta D_{KL} \right] LGRPO​=E[min(πold​(oi​∣q)πθ​(oi​∣q)​Ai​,clip(…)Ai​)−βDKL​]

3. 大模型训练实例推演

为了彻底理解 GRPO，我们通过一个具体的 LLM 续写任务 来演示全过程。

场景设置：

  • Prompt: "天空是"
  • Group Size (GGG): 4
  • Reward Model: 根据语义准确性打分。

阶段一：组采样 (Rollout)

模型基于 Prompt 生成了 4 个不同的回答（Temperature > 0）：

编号 生成的回答 (oio_ioi​) 隐含含义
Output 1 "蓝色的" 正确，简洁
Output 2 "广阔的" 正确，稍次
Output 3 "绿色的" 错误
Output 4 "好吃的" 离谱，幻觉

阶段二：打分与优势计算 (Evaluation)

Reward Model 给出分数：

  • r1=10.0r_1 = 10.0r1​=10.0
  • r2=8.0r_2 = 8.0r2​=8.0
  • r3=−2.0r_3 = -2.0r3​=−2.0
  • r4=−10.0r_4 = -10.0r4​=−10.0

1. 计算基线 (Baseline):

  • Mean = (10+8−2−10)/4=1.5(10+8-2-10) / 4 = 1.5(10+8−2−10)/4=1.5
  • Std (样本标准差) ≈8.38\approx 8.38≈8.38

2. 计算优势 (Advantage):

  • A1A_1A1​ (“蓝色的”): (10−1.5)/8.38≈+1.01(10 - 1.5) / 8.38 \approx \mathbf{+1.01}(10−1.5)/8.38≈+1.01
    • 解读：表现优异，正向激励。
  • A2A_2A2​ (“广阔的”): (8−1.5)/8.38≈+0.77(8 - 1.5) / 8.38 \approx \mathbf{+0.77}(8−1.5)/8.38≈+0.77
    • 解读：表现良好，正向激励。
  • A3A_3A3​ (“绿色的”): (−2−1.5)/8.38≈−0.41(-2 - 1.5) / 8.38 \approx \mathbf{-0.41}(−2−1.5)/8.38≈−0.41
    • 解读：表现较差，轻微惩罚。
  • A4A_4A4​ (“好吃的”): (−10−1.5)/8.38≈−1.37(-10 - 1.5) / 8.38 \approx \mathbf{-1.37}(−10−1.5)/8.38≈−1.37
    • 解读：表现极差，严厉惩罚。

阶段三：参数更新 (Training)

我们将这些 AiA_iAi​ 广播给对应句子中的每一个 Token。

  • 对于生成 “蓝”, “色” 的 Token，梯度方向是 增加概率 (权重 1.01)。
  • 对于生成 “好”, “吃”, “的” 的 Token，梯度方向是 降低概率 (权重 1.37)。

注意：这里完全没有 Critic 参与，基线 1.5 是由这 4 个样本“内卷”得出来的。

4. GRPO 核心代码实现 (PyTorch 风格)

这是 GRPO 计算 Loss 的简化核心代码。可以看到，它比起 PPO 的 GAE 计算要简单得多。

import torch
import torch.nn.functional as F

def grpo_loss(batch_rewards, batch_log_probs, old_log_probs, epsilon=0.2, beta=0.01):
    """
    GRPO Loss 计算
    
    Args:
        batch_rewards: [Batch_Size, Group_Size] - 每一组生成的奖励
        batch_log_probs: [Batch_Size, Group_Size, Seq_Len] - 当前策略的概率
        old_log_probs: [Batch_Size, Group_Size, Seq_Len] - 采样时的旧概率
    """
    
    # 1. 计算优势 (Advantage) - 核心步骤
    # 在 Group 维度 (dim=1) 上计算均值和标准差
    mean = batch_rewards.mean(dim=1, keepdim=True)
    std = batch_rewards.std(dim=1, keepdim=True)
    
    # 归一化得到优势
    # advantages shape: [Batch_Size, Group_Size, 1]
    advantages = (batch_rewards - mean) / (std + 1e-5)
    
    # 2. 计算重要性采样比率 (Ratio)
    # 广播 advantages 到序列长度: [Batch, Group, Seq_Len]
    advantages = advantages.unsqueeze(-1).expand_as(batch_log_probs)
    
    # ratio = exp(new - old)
    ratio = torch.exp(batch_log_probs - old_log_probs)
    
    # 3. PPO Clip Loss
    surr1 = ratio * advantages
    surr2 = torch.clamp(ratio, 1.0 - epsilon, 1.0 + epsilon) * advantages
    policy_loss = -torch.min(surr1, surr2).mean()
    
    # 4. (可选) 计算 KL 惩罚，防止偏离
    # 这里简化处理，通常 KL 是逐 Token 计算并加在 Reward 里的
    # kl_loss = ... 
    
    return policy_loss
