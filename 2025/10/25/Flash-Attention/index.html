Flash-Attention算法

Created2025-10-25|Updated2025-10-27|大模型
|Word Count:5.8k|Reading Time:22mins|Post Views:

前言

学Flash-Attention算法的主要契机是最近在工作中遇到了进行长文本训练时的OOM（Out of Memory）问题，在训练时使用Flash-Attention是尝试过的有效的方法之一，所以就想学一下这个算法的原理。

Paper: https://arxiv.org/pdf/2205.14135

参考：https://www.zhihu.com/question/611236756/answer/3132304304

背景

传统的transformer算法中，对于self-attention的计算是一个比较费时费空间的做法，其存储复杂度是输入文本长度的平方复杂度，所以当输入序列变得很长时训练速度会变得比较慢，且存在OOM的风险，当前许多研究工作着重于减小self-attention的计算复杂度以加快训练速度，减小显存占用。

FlashAttention主要解决Transformer计算速度慢和存储占用高的问题。但与绝大多数Efficient Transformer把改进方法集中在降低模型的FLOPS（floating point operations per second）不同，FlashAttention将优化重点放在了降低存储访问开销（Memory Access Cost，MAC）上。

Self-Attention 机制简介

首先回顾一下Self-Attention的计算方式。

Self-Attention（自注意力）是 Transformer 模型中的核心组件，用于捕捉序列中不同位置之间的依赖关系。其基本思想是：对于序列中的每个元素，通过计算它与其他所有元素的相关性（注意力权重），对整个序列进行加权聚合，从而得到该元素的上下文感知表示。

给定输入序列 $X = [x_1, x_2, \dots, x_n] \in \mathbb{R}^{N \times d}$，其中 $n$ 是序列长度，$d$ 是特征维度。

  1. 计算注意力分数（Attention Scores）：
  • $QK^\top$ 计算所有 Query 与 Key 的点积，得到 $N \times N$ 的相似度矩阵；
  • 除以 $\sqrt{d_k}$ 用于缩放，缓解点积过大导致 softmax 梯度消失的问题；
  • $\text{softmax}$ 沿行归一化，得到注意力权重；
  • 最后与 $V$ 相乘，对 Value 进行加权求和。
  1. 输出：

最终输出是一个与输入序列等长的表示矩阵，每个位置的输出融合了整个序列的上下文信息。

注：在实际应用中，通常使用多头注意力（Multi-Head Attention）来增强模型的表达能力。

正因为Transformer的复杂度随序列长度的增长呈二次方增长，所以通常基于Transformer的大语言模型的上下文长度都不会特别长（如早期的LLM普遍在2k、4k左右）。上下文长度的局限性也限制了大语言模型的应用场景，尤其是对输入长度要求比较高的场景。

SRAM、DRAM 与 HBM 存储器简介

在计算机系统中，存储器根据速度、容量、功耗和成本等特性分为多个层级。SRAM、DRAM 和 HBM 是三种重要的半导体存储技术，广泛应用于 CPU 缓存、主存和高性能计算场景。

1. SRAM（Static Random-Access Memory，静态随机存取存储器）

  • 原理：使用 6 个晶体管（6T）构成一个存储单元，通过双稳态触发器保存数据，无需刷新。
  • 特点：
    • 速度快：访问延迟低（纳秒级），常用于 CPU 的 L1/L2/L3 缓存。
    • 功耗低（静态时）：只要通电，数据就保持不变。
    • 面积大、成本高：每个单元需要多个晶体管，集成密度低。
    • 易失性：断电后数据丢失。

2. DRAM（Dynamic Random-Access Memory，动态随机存取存储器）

  • 原理：每个存储单元由 1 个晶体管和 1 个电容（1T1C）组成，通过电容电荷存储数据，需定期刷新（通常每 64ms 一次）。
  • 特点：
    • 容量大、成本低：单元结构简单，集成密度高，是主流主存（如 DDR4/DDR5）的基础。
    • 速度较慢：访问延迟高于 SRAM（通常几十纳秒）。
    • 功耗较高：因需周期性刷新。
    • 易失性：断电后数据丢失。

3. HBM（High Bandwidth Memory，高带宽存储器）

  • 原理：基于 DRAM 技术，采用 3D 堆叠 + 硅通孔（TSV） 技术，将多个 DRAM 芯片垂直堆叠，并通过中介层（Interposer）与处理器（如 GPU）紧密集成。
  • 特点：
    • 极高带宽：通过数千位宽的并行接口（如 HBM3 带宽可达 1 TB/s 以上）。
    • 低功耗/位：短互连距离降低数据传输能耗。
    • 体积小：堆叠结构节省 PCB 面积。
    • 成本高、容量受限：制造复杂，主要用于高端 GPU、AI 加速器和 HPC 场景。
    • 仍为易失性存储器。

对比总结

特性 SRAM DRAM HBM
单元结构 6T 1T1C 堆叠 DRAM + TSV
速度 最快 中等 高带宽（非低延迟）
密度/容量 低 高 中等（受限于堆叠层数）
成本 高 低 很高
典型用途 CPU 缓存 系统主存（DDR） GPU/AI 芯片的片外高速缓存
易失性 是 是 是

注：HBM 本质上是 DRAM 的一种先进封装形式，旨在突破传统 DDR 内存的带宽瓶颈。

Flash-Attention V1

FlashAttention的核心思路

FlashAttention的目标是降低MAC，即使代价是增加了FLOPS。

下面是论文中对SRAM、HBM和DRAM的简要介绍，Attention的过程主要是围绕SRAM和HBM展开，可以看出SRAM运算速度快，但是存储空间比较小，而HBM则相反。标准Attention存在较多的SRAM和HBM之间的数据交换，而这个过程往往是比较耗时的，而FlashAttention的核心思路则是减少这种数据交换。

标准Transformer的计算可以抽象为如下过程，它主要使用了HBM：

上图中一共包含八次HBM的矩阵读写操作。这八次读写操作分别为：

  1. 计算注意力分数矩阵 S

    • 从 HBM 中分块加载 Query 矩阵 $Q$ 和 Key 矩阵 $K$ 到SRAM（两次）；
    • 计算点积 $S = QK^\top$，得到注意力分数矩阵；
    • 将结果 $S$ 写回 HBM 临时存储（一次）。

  2. 计算注意力权重矩阵 P

    • 从 HBM 读取 $S$ 到SRAM（一次）；
    • 对每行应用 softmax 函数：$P = \text{softmax}(S)$，得到归一化注意力权重；
    • 将 $P$ 写回 HBM（一次）。

  3. 计算输出矩阵 O

    • 从 HBM 分块加载 $P$ 和 Value 矩阵 $V$（两次）；
    • 计算加权和：$O = PV$；
    • 将最终输出 $O$ 写回 HBM 并返回（一次）。

接下来我们计算读写操作的具体复杂度：

动作 操作数据量
读取$Q$和$K$矩阵 $2Nd$
存储和读取$S$矩阵 $2N^2$
存储和读取$P$矩阵 $2N^2$
读取$V$矩阵 $Nd$
存储$O$矩阵 $Nd$

一次计算下来的操作数据量为：$4Nd+4N^2$

FlashAttention 中 Softmax 的动态更新机制（仅 Softmax 部分，显式维护 SM 向量视角）

在理解 FlashAttention 的核心思想时，一种有效的教学视角是：假设我们显式维护一个完整的 Softmax 概率向量 $SM$（对应某一个 query 的注意力权重），并观察它如何随着注意力分数矩阵 $S = qK^\top / \sqrt{d}$ 的分块计算而逐步更新。尽管 FlashAttention 实际实现中并不存储 $SM$，但此视角有助于清晰揭示 Softmax 的在线归一化本质。

为了便于理解，假设每次分块的$Q$矩阵为1×d，$K$和$V$矩阵分块为d×2，那么对应该$Q$矩阵的Softmax的大小为$SM \in \mathbb{R}^{1 \times N}$，每次计算的部分Softmax向量大小为1×2。

1. 问题设定

  • 序列长度为 $N$，特征维度为 $d$。
  • 固定一个 query 向量 $q \in \mathbb{R}^{1 \times d}$。
  • Key 矩阵 $K \in \mathbb{R}^{N \times d}$ 按列（token）分块，每块包含 2 个 token，即块大小 $B = 2$。
  • 每次计算局部注意力分数：对应完整 $S$ 行向量的第 $2t-1$ 到 $2t$ 个元素。
  • 假设我们显式维护一个 $1 \times N$ 的 Softmax 向量 $SM$，初始为空。

初始化

  • 全局最大值：$m = -\infty$
  • 全局缩放指数和：$\ell = 0$
  • $SM = [\,]$（空向量）

第 1 次迭代（处理第 1 块，位置 1–2）

  1. 计算局部分数：$s^{(1)} = [s_1, s_2]$

FlashAttention v1的前向计算过程

本算法是 FlashAttention 的核心实现，其目标是在 不将中间注意力矩阵写入高带宽存储器（HBM） 的前提下，通过分块计算和在线 Softmax 更新，高效完成标准注意力操作：

算法利用片上 SRAM（大小为 $M$）作为高速缓存，仅在必要时从 HBM 加载数据，显著降低内存访问量。

1. 输入与资源要求

  • 输入：三个矩阵 $Q, K, V \in \mathbb{R}^{N \times d}$，存储于 HBM。
  • 硬件资源：片上 SRAM 容量为 $M$（单位：字节数或浮点数个数），用于临时存放分块数据和中间状态。

2. 块尺寸设置（步骤 1）

根据 SRAM 大小 $M$ 设置两个关键块尺寸：

  • 列块大小（对应 Key/Value 列方向）：

    • 解释：每个块 $K_j, V_j$ 大小为 $B_c \times d$，需加载到 SRAM。乘以 4 是因为每步计算涉及 $K, V, O, m, \ell$ 等多个变量，预留空间。

  • 行块大小（对应 Query 行方向）：

    • 解释：每个 Query 块 $Q_i$ 大小为 $B_r \times d$。若 $d$ 很小，则取 $d$ 以避免过小块导致效率下降。

注：$B_c$ 和 $B_r$ 的设计确保每次加载的数据总量不超过 SRAM 容量。

3. 初始化（步骤 2）

在 HBM 中分配并初始化以下全局变量：

  • 输出矩阵 $O = 0 \in \mathbb{R}^{N \times d}$
  • 归一化因子向量 $\ell = 0 \in \mathbb{R}^N$（即每行的缩放指数和）
  • 最大值向量 $m = -\infty \in \mathbb{R}^N$（即每行的最大注意力分数）

这些变量将在后续迭代中被动态更新。

4. 数据分块（步骤 3 & 4）

  • Query 分块：将 $Q$ 沿行划分为 $Tr = \left\lceil \frac{N}{B_r} \right\rceil$ 个块：$Q_1, Q_2, \dots, Q{T_r}$，每个大小为 $B_r \times d$。
  • Key/Value 分块：将 $K, V$ 沿列划分为 $Tc = \left\lceil \frac{N}{B_c} \right\rceil$ 个块：$K_1, \dots, K{Tc}$ 和 $V_1, \dots, V{T_c}$，每个大小为 $B_c \times d$。
  • 输出与状态分块：将 $O, \ell, m$ 同样按行划分为 $T_r$ 个块：$O_i, \ell_i, m_i$，每个大小为 $B_r$（标量向量）。

MAC分析

Flash-Attention V1的MAC主要取决于内循环和外循环：

  • 初始化$Q$矩阵，$\ell$和$m$向量的MAC开销为$Nd+2N$

  • 内循环很简单，一次完整的内循环需要读取完整的 $Q$，MAC开销为$Nd$。

  • 外循环决定了内循环执行的次数，即 $T_c$ 次，$T_c = \left\lceil \frac{N}{B_c} \right\rceil = \left\lceil \frac{4dN}{M} \right\rceil$。可知 FlashAttention 对于$Q$矩阵读取的开销为 $4N^2 d^2 M^{-1}$。

  • $O$矩阵的读取和写入操作与$Q$类似，因为他们同时处于内循环中切其大小相等，区别在于$O$矩阵需要进行读写操作而$Q$矩阵只需要进行读取操作即可，那么$O$矩阵的MAC开销为$8N^2 d^2 M^{-1}$

  • 同理，$\ell$和$m$向量在一次外循环中也需要进行完整的读写操作，所以其MAC开销为$4T_cN=16N^2 d M^{-1}$

  • 外循环中对于$K$矩阵和$V$矩阵的读取操作的MAC开销为$2Nd$

总体MAC开销为：

回顾普通attention的MAC开销：
| 方法 | 总 MAC 表达式 |
|———|————————|
| Standard Attention | $MAC{standard} = 4N^2 + 4Nd$|
| FlashAttention V1 | $MAC{flash} = 3Nd + 2N+\frac{12N^2 d^2+16N^2 d}{M}$|

为了比较二者关系，这里定义$y = MAC{standard} / MAC{flash}$，假设$M = 192KB$，绘制$y-N-d$的函数关系图：

可以看出在一般情况下（$d<=128$），$MAC{standard}$总是大于$MAC{flash}$。

在 Transformer 模型的实际应用场景中，需结合 head 维度设计逻辑、工程优化手段 和 模型效果约束，判断 FlashAttention 相对标准注意力的 MAC（HBM 访问开销）优势是否持续。即使头维度 $d$ 增大至 256（主流模型的单 head 维度上限附近），核心结论仍“基本成立”，具体分析如下：

1. 典型大 $d$ 场景（$d=256$）：MAC 接近但未被超越，工程优化可回归优势

以主流序列长度 $N=1024$、数据类型 FP16、GPU SRAM 容量 $M=192\,\text{KB}$ 为基准，对比两种注意力的 MAC可见，$d=256$ 时两者 MAC 高度接近，但 FlashAttention 未被超越；且工程中会通过 拆分 head（如将 $d=256$ 拆分为 4 个 $d=64$ 的 head）回归典型场景，此时 FlashAttention 的 MAC 会重新降至标准注意力的 15% 以下（符合论文优化效果）。

2. 极端大 $d$ 场景（$d>1000$）：结论“不成立”但无实际应用价值

仅当 $d$ 增大至“突破 Transformer 设计逻辑”的极端值（如 $d=24576$，FP16 下需 SRAM 容纳单 token 数据）时，FlashAttention 的 MAC 才可能超过标准注意力，但这种场景存在两大致命问题：

  • 违背多头注意力设计原则：
    Transformer 的“多头注意力”核心是将大维度 $d{\text{model}}$ 拆分为多个小 head（如 $d{\text{model}}=1024$ 拆分为 16 个 $d=64$ 的 head），单 head 维度过大（$d>256$）会导致“注意力分散”——每个 head 无法捕捉局部语义关联，模型效果严重下降（实验表明，单 head $d=64$ 时模型困惑度最低）。

  • 脱离硬件实际约束：
    现代 GPU 的 SRAM 容量多为 64KB~256KB（如 A100 单流多处理器 SRAM 为 192KB），$d>1000$ 时块尺寸会被压缩至 $B_c<10$（单个列块仅含 10 个 token），分块数量 $T_c>100$，频繁的 HBM 读写会引发“分块 overhead”，但这种场景无任何主流模型采用（目前 GPT-4、LLaMA 3 等模型的单 head 维度均 $\leq 128$）。

⭐3. 论文强调 FlashAttention 的优势，主要不是因为总 MAC 更小，而是因为：

  • 避免 $O(N^2)$ 显存占用

    • Standard Attention 需要同时在 HBM 中存储 $S$ 和 $P$（共 $2N^2$），当 $N=8192$ 时，仅此就需 ~512 MB（单头），多头叠加后极易 OOM。
    • FlashAttention 的峰值显存为 $O(Nd)$，可支持更长序列。

  • 访存模式更规则、可流水

    • Standard Attention 的 $N^2$ 写入是突发的、非结构化的，带宽利用率低。
    • FlashAttention 的块访问适合 GPU 的内存层级和并行调度。

  • 实际运行时间由带宽瓶颈决定，而非总 MAC

    • 即使总 MAC 略高，但因避免了大矩阵写入/读取的带宽争用，实际延迟更低。
