Flash-Attention算法

Created2025-10-25|Updated2025-10-27|大模型
|Word Count:6.9k|Reading Time:28mins|Post Views:

前言

学Flash-Attention算法的主要契机是最近在工作中遇到了进行长文本训练时的OOM（Out of Memory）问题，在训练时使用Flash-Attention是尝试过的有效的方法之一，所以就想学一下这个算法的原理。

Paper: https://arxiv.org/pdf/2205.14135

参考：https://www.zhihu.com/question/611236756/answer/3132304304

背景

传统的transformer算法中，对于self-attention的计算是一个比较费时费空间的做法，其存储复杂度是输入文本长度的平方复杂度，所以当输入序列变得很长时训练速度会变得比较慢，且存在OOM的风险，当前许多研究工作着重于减小self-attention的计算复杂度以加快训练速度，减小显存占用。

FlashAttention主要解决Transformer计算速度慢和存储占用高的问题。但与绝大多数Efficient Transformer把改进方法集中在降低模型的FLOPS（floating point operations per second）不同，FlashAttention将优化重点放在了降低存储访问开销（Memory Access Cost，MAC）上。

Self-Attention 机制简介

首先回顾一下Self-Attention的计算方式。

Self-Attention（自注意力）是 Transformer 模型中的核心组件，用于捕捉序列中不同位置之间的依赖关系。其基本思想是：对于序列中的每个元素，通过计算它与其他所有元素的相关性（注意力权重），对整个序列进行加权聚合，从而得到该元素的上下文感知表示。

给定输入序列 X=[x1,x2,…,xn]∈RN×dX = [x_1, x_2, \dots, x_n] \in \mathbb{R}^{N \times d}X=[x1​,x2​,…,xn​]∈RN×d，其中 nnn 是序列长度，ddd 是特征维度。

  1. 线性变换得到 Query、Key、Value：

Q=XWQ∈RN×dk,K=XWK∈RN×dk,V=XWV∈RN×dv,\begin{aligned} Q &= X W_Q \in \mathbb{R}^{N \times d_k}, \\ K &= X W_K \in \mathbb{R}^{N \times d_k}, \\ V &= X W_V \in \mathbb{R}^{N \times d_v}, \end{aligned} QKV​=XWQ​∈RN×dk​,=XWK​∈RN×dk​,=XWV​∈RN×dv​,​

其中 WQ∈Rd×dkW_Q \in \mathbb{R}^{d \times d_k}WQ​∈Rd×dk​、WK∈Rd×dkW_K \in \mathbb{R}^{d \times d_k}WK​∈Rd×dk​、WV∈Rd×dvW_V \in \mathbb{R}^{d \times d_v}WV​∈Rd×dv​ 是可学习的参数矩阵。

  1. 计算注意力分数（Attention Scores）：

Attention(Q,K,V)=softmax(QK⊤dk)V\text{Attention}(Q, K, V) = \text{softmax}\left( \frac{QK^\top}{\sqrt{d_k}} \right) V Attention(Q,K,V)=softmax(dk​​QK⊤​)V

  • QK⊤QK^\topQK⊤ 计算所有 Query 与 Key 的点积，得到 N×NN \times NN×N 的相似度矩阵；
  • 除以 dk\sqrt{d_k}dk​​ 用于缩放，缓解点积过大导致 softmax 梯度消失的问题；
  • softmax\text{softmax}softmax 沿行归一化，得到注意力权重；
  • 最后与 VVV 相乘，对 Value 进行加权求和。
  1. 输出：

最终输出是一个与输入序列等长的表示矩阵，每个位置的输出融合了整个序列的上下文信息。

注：在实际应用中，通常使用多头注意力（Multi-Head Attention）来增强模型的表达能力。

正因为Transformer的复杂度随序列长度的增长呈二次方增长，所以通常基于Transformer的大语言模型的上下文长度都不会特别长（如早期的LLM普遍在2k、4k左右）。上下文长度的局限性也限制了大语言模型的应用场景，尤其是对输入长度要求比较高的场景。

SRAM、DRAM 与 HBM 存储器简介

在计算机系统中，存储器根据速度、容量、功耗和成本等特性分为多个层级。SRAM、DRAM 和 HBM 是三种重要的半导体存储技术，广泛应用于 CPU 缓存、主存和高性能计算场景。

1. SRAM（Static Random-Access Memory，静态随机存取存储器）

  • 原理：使用 6 个晶体管（6T）构成一个存储单元，通过双稳态触发器保存数据，无需刷新。
  • 特点：
    • 速度快：访问延迟低（纳秒级），常用于 CPU 的 L1/L2/L3 缓存。
    • 功耗低（静态时）：只要通电，数据就保持不变。
    • 面积大、成本高：每个单元需要多个晶体管，集成密度低。
    • 易失性：断电后数据丢失。

2. DRAM（Dynamic Random-Access Memory，动态随机存取存储器）

  • 原理：每个存储单元由 1 个晶体管和 1 个电容（1T1C）组成，通过电容电荷存储数据，需定期刷新（通常每 64ms 一次）。
  • 特点：
    • 容量大、成本低：单元结构简单，集成密度高，是主流主存（如 DDR4/DDR5）的基础。
    • 速度较慢：访问延迟高于 SRAM（通常几十纳秒）。
    • 功耗较高：因需周期性刷新。
    • 易失性：断电后数据丢失。

3. HBM（High Bandwidth Memory，高带宽存储器）

  • 原理：基于 DRAM 技术，采用 3D 堆叠 + 硅通孔（TSV） 技术，将多个 DRAM 芯片垂直堆叠，并通过中介层（Interposer）与处理器（如 GPU）紧密集成。
  • 特点：
    • 极高带宽：通过数千位宽的并行接口（如 HBM3 带宽可达 1 TB/s 以上）。
    • 低功耗/位：短互连距离降低数据传输能耗。
    • 体积小：堆叠结构节省 PCB 面积。
    • 成本高、容量受限：制造复杂，主要用于高端 GPU、AI 加速器和 HPC 场景。
    • 仍为易失性存储器。

对比总结

单元结构 6T 1T1C 堆叠 DRAM + TSV
速度 最快 中等 高带宽（非低延迟）
密度/容量 低 高 中等（受限于堆叠层数）
成本 高 低 很高
典型用途 CPU 缓存 系统主存（DDR） GPU/AI 芯片的片外高速缓存
易失性 是 是 是

注：HBM 本质上是 DRAM 的一种先进封装形式，旨在突破传统 DDR 内存的带宽瓶颈。

Flash-Attention V1

FlashAttention的核心思路

FlashAttention的目标是降低MAC，即使代价是增加了FLOPS。

下面是论文中对SRAM、HBM和DRAM的简要介绍，Attention的过程主要是围绕SRAM和HBM展开，可以看出SRAM运算速度快，但是存储空间比较小，而HBM则相反。标准Attention存在较多的SRAM和HBM之间的数据交换，而这个过程往往是比较耗时的，而FlashAttention的核心思路则是减少这种数据交换。

标准Transformer的计算可以抽象为如下过程，它主要使用了HBM：

上图中一共包含八次HBM的矩阵读写操作。这八次读写操作分别为：

  1. 计算注意力分数矩阵 S

    • 从 HBM 中分块加载 Query 矩阵 QQQ 和 Key 矩阵 KKK 到SRAM（两次）；
    • 计算点积 S=QK⊤S = QK^\topS=QK⊤，得到注意力分数矩阵；
    • 将结果 SSS 写回 HBM 临时存储（一次）。

  2. 计算注意力权重矩阵 P

    • 从 HBM 读取 SSS 到SRAM（一次）；
    • 对每行应用 softmax 函数：P=softmax(S)P = \text{softmax}(S)P=softmax(S)，得到归一化注意力权重；
    • 将 PPP 写回 HBM（一次）。

  3. 计算输出矩阵 O

    • 从 HBM 分块加载 PPP 和 Value 矩阵 VVV（两次）；
    • 计算加权和：O=PVO = PVO=PV；
    • 将最终输出 OOO 写回 HBM 并返回（一次）。

Softmax的动态更新

FlashAttention 通过 在线（online）或流式（streaming）Softmax 技术，在不存储完整注意力分数矩阵的前提下，动态维护 Softmax 的归一化因子和最大值。该机制仅关注如何从分块的输入 StS_tSt​（即局部注意力分数）逐步构建全局 Softmax 结果，不涉及输出 O=PVO = PVO=PV 的计算。

  • 此流程 精确等价于一次性计算完整 Softmax，因为每一步都保持全局归一化。
  • 实际 FlashAttention 不存储 SMSMSM，而是将上述逻辑与 VVV 相乘融合，直接累积输出 OOO。
  • 但此“显式 SM”视角 清晰展示了 Softmax 如何在分块输入下动态重归一化，是理解在线 Softmax 机制的有效途径。
  • ℹ️也就是说，每次循环会计算1×2的Softmax向量，但是会更新之前循环的所有Softmax值，上述公式是对“Softmax值更新”中的公式的运用，主要是从向量更新的角度说明了SMSMSM向量的逐步更新逻辑

MAC分析

Flash-Attention V1的MAC主要取决于内循环和外循环：

  • 初始化QQQ矩阵，ℓ\ellℓ和mmm向量的MAC开销为Nd+2NNd+2NNd+2N

  • 内循环很简单，一次完整的内循环需要读取完整的 QQQ，MAC开销为NdNdNd。

  • 外循环决定了内循环执行的次数，即 TcT_cTc​ 次，Tc=⌈NBc⌉=⌈4dNM⌉T_c = \left\lceil \frac{N}{B_c} \right\rceil = \left\lceil \frac{4dN}{M} \right\rceilTc​=⌈Bc​N​⌉=⌈M4dN​⌉。可知 FlashAttention 对于QQQ矩阵读取的开销为 4N2d2M−14N^2 d^2 M^{-1}4N2d2M−1。

  • OOO矩阵的读取和写入操作与QQQ类似，因为他们同时处于内循环中切其大小相等，区别在于OOO矩阵需要进行读写操作而QQQ矩阵只需要进行读取操作即可，那么OOO矩阵的MAC开销为8N2d2M−18N^2 d^2 M^{-1}8N2d2M−1

  • 同理，ℓ\ellℓ和mmm向量在一次外循环中也需要进行完整的读写操作，所以其MAC开销为4TcN=16N2dM−14T_cN=16N^2 d M^{-1}4Tc​N=16N2dM−1

  • 外循环中对于KKK矩阵和VVV矩阵的读取操作的MAC开销为2Nd2Nd2Nd

在 Transformer 模型的实际应用场景中，需结合 head 维度设计逻辑、工程优化手段 和 模型效果约束，判断 FlashAttention 相对标准注意力的 MAC（HBM 访问开销）优势是否持续。即使头维度 ddd 增大至 256（主流模型的单 head 维度上限附近），核心结论仍“基本成立”，具体分析如下：

1. 典型大 ddd 场景（d=256d=256d=256）：MAC 接近但未被超越，工程优化可回归优势

以主流序列长度 N=1024N=1024N=1024、数据类型 FP16、GPU SRAM 容量 M=192 KBM=192\,\text{KB}M=192KB 为基准，对比两种注意力的 MAC可见，d=256d=256d=256 时两者 MAC 高度接近，但 FlashAttention 未被超越；且工程中会通过 拆分 head（如将 d=256d=256d=256 拆分为 4 个 d=64d=64d=64 的 head）回归典型场景，此时 FlashAttention 的 MAC 会重新降至标准注意力的 15% 以下（符合论文优化效果）。

2. 极端大 ddd 场景（d>1000d>1000d>1000）：结论“不成立”但无实际应用价值

仅当 ddd 增大至“突破 Transformer 设计逻辑”的极端值（如 d=24576d=24576d=24576，FP16 下需 SRAM 容纳单 token 数据）时，FlashAttention 的 MAC 才可能超过标准注意力，但这种场景存在两大致命问题：

  • 违背多头注意力设计原则：
    Transformer 的“多头注意力”核心是将大维度 dmodeld_{\text{model}}dmodel​ 拆分为多个小 head（如 dmodel=1024d_{\text{model}}=1024dmodel​=1024 拆分为 16 个 d=64d=64d=64 的 head），单 head 维度过大（d>256d>256d>256）会导致“注意力分散”——每个 head 无法捕捉局部语义关联，模型效果严重下降（实验表明，单 head d=64d=64d=64 时模型困惑度最低）。

  • 脱离硬件实际约束：
    现代 GPU 的 SRAM 容量多为 64KB~256KB（如 A100 单流多处理器 SRAM 为 192KB），d>1000d>1000d>1000 时块尺寸会被压缩至 Bc<10B_c<10Bc​<10（单个列块仅含 10 个 token），分块数量 Tc>100T_c>100Tc​>100，频繁的 HBM 读写会引发“分块 overhead”，但这种场景无任何主流模型采用（目前 GPT-4、LLaMA 3 等模型的单 head 维度均 ≤128\leq 128≤128）。

⭐3. 论文强调 FlashAttention 的优势，主要不是因为总 MAC 更小，而是因为：

  • 避免 O(N2)O(N^2)O(N2) 显存占用

    • Standard Attention 需要同时在 HBM 中存储 SSS 和 PPP（共 2N22N^22N2），当 N=8192N=8192N=8192 时，仅此就需 ~512 MB（单头），多头叠加后极易 OOM。
    • FlashAttention 的峰值显存为 O(Nd)O(Nd)O(Nd)，可支持更长序列。

  • 访存模式更规则、可流水

    • Standard Attention 的 N2N^2N2 写入是突发的、非结构化的，带宽利用率低。
    • FlashAttention 的块访问适合 GPU 的内存层级和并行调度。

  • 实际运行时间由带宽瓶颈决定，而非总 MAC

    • 即使总 MAC 略高，但因避免了大矩阵写入/读取的带宽争用，实际延迟更低。
